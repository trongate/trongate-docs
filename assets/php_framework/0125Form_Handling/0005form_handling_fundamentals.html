Form Handling Fundamentals

Forms are the backbone of web applications. Login screens, user profiles, checkout pages, admin panels - they all need forms. And they all need the same things: clean HTML, solid validation, security, and a smooth user experience.

Trongate gives you all of that with zero abstraction.

The Trongate Approach

No form builders. No XML config files. No "magic" methods that generate fifty lines of HTML you never asked for.

Just Native PHP functions that output exactly what you tell them to - and nothing more.

What Trongate Gives You For Form Handling

  • Form helpers - functions like form_input(), form_dropdown(), and form_checkbox() that generate clean HTML
  • Validation module - set rules, run checks, display errors automatically
  • CSRF protection - built-in, automatic, zero setup required
  • Post-Redirect-Get pattern - prevents duplicate submissions, works with flashdata for success messages
  • Session-based error storage - validation errors survive redirects and display exactly where you want them

A Simple Example

Here's a complete working form in Trongate - create, validate, save, redirect:

[code=php] // Controller method public function create(): void { $data['headline'] = 'Create New Task'; $data['form_location'] = BASE_URL.'tasks/submit'; $this->templates->admin($data); } public function submit(): void { $this->validation->set_rules('task_title', 'task title', 'required|min_length[2]'); $this->validation->set_rules('task_description', 'description', 'required'); $result = $this->validation->run(); if ($result === true) { $data['task_title'] = post('task_title', true); $data['task_description'] = post('task_description', true); $new_id = $this->db->insert($data, 'tasks'); set_flashdata('Task successfully created'); redirect('tasks/show/'.$new_id); } else { $this->create(); // Redisplay form with errors } } [/code]

And the view file:

[code=php] <h1><?= $headline ?></h1> <?= validation_errors() ?> <?php echo form_open($form_location); echo form_label('Task Title'); echo form_input('task_title', post('task_title', true)); echo form_label('Description'); echo form_textarea('task_description', post('task_description', true)); echo form_submit('submit', 'Create Task'); echo form_close(); ?> [/code]

That's it. No configuration. No registration. No twenty-step tutorial.

How It Works

  1. User visits the form → create() method displays the view
  2. User submits → submit() validates the data
  3. Validation fails? → Errors stored in session, form redisplays with error messages
  4. Validation passes? → Data saved, flashdata set, user redirected to success page

The Post-Redirect-Get Pattern

When a form submits successfully, Trongate redirects instead of rendering directly. This prevents the dreaded "resubmit form?" browser warning when users hit refresh.

Here's the flow:

  • POST → User submits form data
  • Process → Validate, save to database
  • Redirect → Send user to a new page (via redirect())
  • GET → Display success message (via flashdata())

Success messages survive the redirect using flashdata. Validation errors work the same way - they are stored in the session, displayed after redirect, then automatically cleared.

Security Built In

Every form that uses form_close() automatically includes a CSRF token. Every form submission automatically validates it. If the token is missing or invalid, the request gets blocked. See Form Handling → CSRF Protection for details.

You don't opt in. You don't configure it. It just works.
