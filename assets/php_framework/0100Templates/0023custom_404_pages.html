[meta-description]Customize your 404 Pages in Trongate by changing the ERROR_404 constant in config/config.php. This constant acts as a single source of truth, pointing to any module/method you like (e.g., templates/error_404 or errors/not_found) for total control over the page content and template.[/meta-description]

Custom 404 Pages

Trongate has always made 404 pages dead simple. In v2 it's even cleaner.

The Single Source of Truth

Open this file →

[code]config/config.php[/code]

And you'll find this line →

[code=php] define('ERROR_404', 'templates/error_404'); [/code]

When a URL cannot be matched, Trongate automatically calls →

[code=php] $this->templates->error_404(); [/code]

The above code simply loads this view →

[code]modules/templates/views/error_404.php[/code]

It uses the exact same mechanism as admin() and public().

Three Ways to Customize Your 404 Pages

Option 1 – Quick & Easy (most common)

Just edit the existing file →

[code]modules/templates/views/error_404.php[/code]

Make it friendly, funny, or brutally honest – your call.

Option 2 – Use a Different View in the templates Module

Add a new method to this file →

[code]modules/templates/Templates.php[/code] [code=php] public function my_cool_404(array $data = []): void { $this->display('my_cool_404', $data); } [/code]

Create this view →

[code]modules/templates/views/my_cool_404.php[/code]

Then change the constant →

[code=php] define('ERROR_404', 'templates/my_cool_404'); [/code]

Option 3 – Point to Any Module/Method You Like

Want a completely custom 404 page with extra logic? Point the constant anywhere →

[code=php] define('ERROR_404', 'errors/not_found'); // or define('ERROR_404', 'welcome/oops'); // or even define('ERROR_404', 'modern_templates/minimal'); [/code]

Trongate will automatically call that module and method.

Real-World Example (Simple & Clean)

Here's a tidy, beginner-friendly custom 404 that still feels professional →

[code=php] // config/config.php define('ERROR_404', 'errors/show_404'); [/code]

Then, from within modules/errors/Errors.php:

[code=php] <?php class Errors extends Trongate { public function show_404(array $data = []): void { $data['page_title'] = 'Page Not Found'; $data['message'] = 'Sorry, we couldn't find that page.'; $this->templates->public($data); } } [/code]

You can, of course, make it as fancy as you like later – add logging, suggested pages, a search box – but the pattern is always the same: one constant, one method, one template.

One line in config.php → total control over every 404.

No hooks, no events, no middleware. Just Native PHP doing what it does best.
