Installation & Configuration

After downloading Trongate, installation takes only a few minutes. There are no dependency managers, no build steps, and no scaffolding commands. Just upload and configure.

1. Place the Files on Your Server

Upload the contents of the Trongate ZIP file to your web server's document root. On many shared hosts this is a directory called public_html or www. The resulting structure should look like this:

[code] your-site.com/ ├── config/ ├── engine/ ├── modules/ ├── public/ ├── uploads/ ├── .htaccess └── index.php [/code]

The public/ directory is the web root. Point your domain to public/ if possible — this keeps the framework files one level above public access for better security.

2. Configure Your Base URL

Open config/config.php and set the BASE_URL constant to the URL where your application will be accessed. Also, set ENV to 'dev' for development or 'prod' (or 'live') for production:

[code=php] define('BASE_URL', 'http://localhost/trongate/'); define('ENV', 'dev'); [/code]

Warning: Never launch a Trongate application into a live or production environment with the ENV constant set to 'dev' or 'DEV'.

When ENV is set to 'dev', the framework may display detailed error messages and debugging information that could expose sensitive details about your application. For production, always set ENV to 'prod', 'live', or any value other than 'dev'.

3. Configure Your Database

Open config/database.php and enter your database connection details:

[code=php] <?php $databases = [ 'default' => [ 'host' => '127.0.0.1', 'port' => '3306', 'user' => 'root', 'password' => '', 'database' => 'trongate' ] ];[/code]

You can define multiple database groups by adding more entries to the $databases array. Each group can then be accessed from your models using the group name as a property (e.g., $this->analytics).

4. Verify the Installation

Navigate to the URL you set in BASE_URL. You should see the default Trongate welcome page. If you see a blank page or an error, check that PHP 8.0+ is installed and that your BASE_URL is correct.

That’s it. You’re ready to build.
