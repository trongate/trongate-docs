form_open()

function form_open(string $location, array $attributes = []): string

Description

Generates the opening tag for an HTML form. This is typically the first function called when building a form in Trongate.

Parameters

ParameterTypeDescription
$locationstringThe URL or path where the form will be submitted.
$attributesarray(optional) HTML attributes for the form element. Defaults to an empty array ([]).

Return Value

TypeDescription
stringThe HTML opening tag for the form.

Example #1: Basic Form

[code=php] echo form_open('users/submit'); echo form_input('username', ''); echo form_submit('submit', 'Save'); echo form_close(); // Output: // &lt;form action="https://example.com/users/submit" method="post"&gt; // &lt;input type="text" name="username"&gt; // &lt;button type="submit" name="submit"&gt;Save&lt;/button&gt; // &lt;/form&gt; [/code]

Example #2: Form with GET Method

[code=php] $attributes = [ 'method' => 'get', 'id' => 'search-form' ]; echo form_open('products/search', $attributes); // Output: // &lt;form action="https://example.com/products/search" method="get" id="search-form"&gt; [/code]

Example #3: Form with Automatic Error Highlighting

[code=php] $attributes = [ 'class' => 'highlight-errors' ]; echo form_open('users/register', $attributes); // Output: // &lt;form action="https://example.com/users/register" method="post" class="highlight-errors"&gt; [/code]

Example #4: Trongate MX Form

[code=php] $attributes = [ 'mx-post' => 'comments/create', 'mx-target' => '#comment-list', 'mx-swap' => 'beforeend' ]; echo form_open('#', $attributes); // Output: // &lt;form mx-post="comments/create" mx-target="#comment-list" mx-swap="beforeend" action="#" method="post"&gt; [/code]

Regarding Trongate MX: When using MX attributes like mx-post, the form submits asynchronously without page reloads. The mx-post attribute takes precedence over the action attribute.

Additional guidance on how to work with Trongate MX is available from the Trongate MX Documentation.

URL Handling

The $location parameter supports different URL formats:

[code=php] // Relative path (prepended with BASE_URL) echo form_open('products/create'); // Output: &lt;form action="https://example.com/products/create" method="post"&gt; // Absolute path echo form_open('/admin/users'); // Output: &lt;form action="/admin/users" method="post"&gt; // Full URL echo form_open('https://api.example.com/submit'); // Output: &lt;form action="https://api.example.com/submit" method="post"&gt; [/code]
  • Always pair form_open() with form_close() to ensure proper form structure and CSRF protection.
  • The default method is post. Specify ['method' => 'get'] for GET forms.
  • For file uploads, use form_open_upload() instead of form_open().
  • Relative paths are automatically converted to full URLs using the BASE_URL constant.
  • All attributes are automatically HTML-escaped for security.
  • The method attribute is handled specially - it's extracted from the attributes array and placed in the proper position.
