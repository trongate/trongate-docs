flashdata()

function flashdata(?string $opening_html = null, ?string $closing_html = null): ?string

Description

Retrieves and returns a one-time flash message from the session, wrapped in configurable HTML. If a message was previously stored via set_flashdata(), this function returns it and immediately clears the session variable so the message is never displayed again. If no message exists, null is returned.

The function wraps Modules::run('flashdata/flashdata', [...]) internally, converting the two parameters into the associative array expected by the module.

The method handles the following behaviour:

  • If no flash message exists in the session, null is returned — no output is generated.
  • The message is automatically cleared from the session upon retrieval, ensuring it displays only once.
  • Custom HTML wrapping can be provided via the $opening_html and $closing_html parameters for one-off overrides.
  • If no custom wrapping is provided, the function checks for globally defined FLASHDATA_OPEN and FLASHDATA_CLOSE constants.
  • If no constants are defined either, a default green <p> tag is used as a fallback.

Parameters

ParameterTypeDescriptionDefault
$opening_html ?string Optional HTML opening tag(s) to wrap the message. For example: '<div class="alert alert-success">'. When null, the function falls back to the FLASHDATA_OPEN constant, or a default '<p style="color: green;">' tag. null
$closing_html ?string Optional HTML closing tag(s) to wrap the message. For example: '</div>'. When null, the function falls back to the FLASHDATA_CLOSE constant, or a default '</p>' tag. null

Return Value

TypeDescription
?stringThe formatted flash message string wrapped in HTML, or null if no flash message exists in the session.

Example #1: Basic Usage

Simply call flashdata() with no arguments to display the message with default formatting.

[code=php] // In your view file echo flashdata(); [/code]

If a message was set via set_flashdata('Record created successfully'), this outputs something like:

<p style="color: green;">Record created successfully</p>

Example #2: Custom HTML Wrapping

Pass custom opening and closing HTML to match your application's design system.

[code=php] echo flashdata( '<div class="alert alert-success" role="alert">', '</div>' ); [/code]

Example #3: With an Error Alert

Different wrapping for different message types.

[code=php] // In your controller set_flashdata('Please check your input and try again.'); // In your view echo flashdata( '<div class="alert alert-danger" role="alert">', '</div>' ); [/code]

Example #4: Using Global Defaults via Config Constants

Define FLASHDATA_OPEN and FLASHDATA_CLOSE in your config/config.php file to set a global default style for all flash messages across your application.

[code=php] // In config/config.php define('FLASHDATA_OPEN', '<div class="toast toast-success">'); define('FLASHDATA_CLOSE', '</div>'); [/code]

With these constants defined, calls to flashdata() with no arguments will automatically use your custom wrapping.

Notes

  • One-time only: The message is automatically removed from the session after retrieval. Refreshing the page will not show the message again.
  • Customisation hierarchy: Function arguments take precedence over config constants (FLASHDATA_OPEN / FLASHDATA_CLOSE), which take precedence over the default green <p> tag.
  • Null-safe: It is safe to call flashdata() at any time — whether or not a message exists. If no message is present, null is returned and no output is generated.
  • Paired with set_flashdata(): This function is designed to work with set_flashdata(). Call set_flashdata() before a redirect, then call flashdata() in the destination view.
  • Module equivalent: flashdata($open, $close) is shorthand for Modules::run('flashdata/flashdata', ['opening_html' => $open, 'closing_html' => $close]).
  • Signature difference: Unlike the module method which accepts an associative array, this helper accepts two distinct string parameters for convenience and readability.
