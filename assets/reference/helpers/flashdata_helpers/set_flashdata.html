set_flashdata()

function set_flashdata(string $msg): void

Description

Stores a one-time flash message into the session. This is the first half of the flash data pattern — call this function before a redirect, and the message persists in the session until it is retrieved by flashdata() on the next page load.

The function wraps Modules::run('flashdata/set_flashdata', $msg) internally, providing a convenient shorthand.

Parameters

ParameterTypeDescriptionDefault
$msg string The message text to store. This is typically a human-readable notification such as 'Record created successfully' or 'Changes saved'. The string is stored as-is without any escaping or transformation. N/A

Return Value

TypeDescription
voidThis function does not return a value. It stores the message directly in the PHP session via the flashdata module.

Example #1: Basic Usage After Form Submission

The most common pattern is to call set_flashdata() after a successful form submission, then redirect to another page where the message will be displayed.

[code=php] public function submit(): void { if (post('submit', true) === 'Submit') { $data = $this->model->get_post_data_for_database(); $this->model->create_new_record($data); set_flashdata('Record created successfully'); redirect('tasks/manage'); } } [/code]

Example #2: Error Notification

Flash messages are also useful for failure notifications.

[code=php] public function delete(int $id): void { $result = $this->model->delete_record($id); if ($result === false) { set_flashdata('Unable to delete record - it may be linked to other data'); } else { set_flashdata('Record deleted successfully'); } redirect('tasks/manage'); } [/code]

Notes

  • One at a time: Only a single flash message can be stored. A subsequent call overwrites any existing message.
  • Redirect-friendly: Flash data survives redirects because it is stored in the session, not in request data.
  • No formatting: The message is stored as plain text. Use the flashdata() helper to apply HTML wrapping when displaying.
  • Paired with flashdata(): This function is designed to work with flashdata() — call set_flashdata() before a redirect, then call flashdata() in the destination view.
  • Module equivalent: set_flashdata('message') is shorthand for Modules::run('flashdata/set_flashdata', 'message').
