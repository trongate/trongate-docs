flashdata()

public function flashdata(array $data = []): ?string

Description

Retrieves and returns a one-time flash message from the session. If a message was previously stored via set_flashdata(), this method returns it wrapped in HTML and immediately clears the session variable so the message is never displayed again.

The method handles the following behaviour:

  • If no flash message exists in the session, null is returned — no output is generated.
  • The message is automatically cleared from the session upon retrieval, ensuring it displays only once.
  • Custom HTML wrapping can be provided via the $data array for one-off overrides.
  • If no custom wrapping is provided, the module checks for globally defined FLASHDATA_OPEN and FLASHDATA_CLOSE constants.
  • If no constants are defined either, a default green <p> tag is used as a fallback.

Parameters

Parameter Type Description
$data array An optional associative array controlling the HTML wrapping of the message. See the $data Keys section below for available keys.

$data Keys

Key Type Description Default Required
opening_html string|null The HTML string to prepend before the message content. For example: '<div class="alert alert-success">'. When null, the module falls back to the FLASHDATA_OPEN constant, or a default '<p style="color: green;">' tag. null No
closing_html string|null The HTML string to append after the message content. For example: '</div>'. When null, the module falls back to the FLASHDATA_CLOSE constant, or a default '</p>' tag. null No

Return Value

Type Description
string|null string — The formatted message with HTML wrapping, ready for output.
null — No flash message was found in the session.

Example #1: Basic Usage with Helper Function

In your view file, simply call flashdata() with no arguments to display the message with default formatting.

<?php
// In your view file
flashdata();
?>

This outputs something like <p style="color: green;">Record created successfully</p> if a message exists, or nothing if there is no message.

Example #2: Custom HTML Wrapping

Pass custom opening and closing HTML to match your application's design system.

<?php
flashdata([
    'opening_html' => '<div class="alert alert-success" role="alert">',
    'closing_html' => '</div>'
]);
?>

Example #3: Using Global Defaults via Config Constants

Define FLASHDATA_OPEN and FLASHDATA_CLOSE in your config/config.php file to set a global default style for all flash messages across your application.

<?php
// In config/config.php
define('FLASHDATA_OPEN', '<div class="toast toast-success">');
define('FLASHDATA_CLOSE', '</div>');
?>

With these constants defined, calls to flashdata() with no arguments will automatically use your custom wrapping.

Example #4: Using the Module Directly

For explicit routing, call the module method via Modules::run(). This is equivalent to the helper function but gives you direct access to the module method.

<?php
$message = Modules::run('flashdata/flashdata', [
    'opening_html' => '<div class="custom-notification">',
    'closing_html' => '</div>'
]);

if ($message !== null) {
    echo $message;
}
?>

Notes

  • One-time only: The message is automatically removed from the session after retrieval. Refreshing the page will not show the message again.
  • Customisation hierarchy: Function arguments take precedence over config constants, which take precedence over the default green <p> tag.
  • Null-safe: It is safe to call flashdata() at any time — whether or not a message exists. If no message is present, null is returned and no output is generated.
  • Paired with set_flashdata(): This method is designed to work in tandem with set_flashdata(). Call set_flashdata() before a redirect, then call flashdata() in the destination view.
