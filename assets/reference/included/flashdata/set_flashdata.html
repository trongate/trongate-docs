set_flashdata()

public function set_flashdata(string $msg): void

Description

Stores a one-time flash message into the session. This is the "set" half of the flashdata pattern — you call this method before a redirect, and the message persists in the session until it is retrieved by flashdata() on the next page load.

The method handles the following behaviour:

  • The message is stored in $_SESSION['flashdata'] as a plain string.
  • Only one flash message can be stored at a time — a subsequent call overwrites any existing message.
  • The message persists across a single redirect and is automatically cleared once retrieved.
  • If the user refreshes the page without calling flashdata(), the message persists in the session.

Parameters

Parameter Type Description Default Required
$msg string The message text to store. This is typically a human-readable notification such as 'Record created successfully' or 'Changes saved'. The string is stored as-is without any escaping or transformation. N/A Yes

Return Value

Type Description
void This method does not return a value. It stores the message directly in the PHP session.

Example #1: Basic Usage with Helper Function

The most common pattern is to call set_flashdata() after a successful form submission, then redirect to another page where the message will be displayed.

<?php
public function submit(): void {
    $this->trongate_security->make_sure_allowed();

    if (post('submit', true) === 'Submit') {
        $data = $this->model->get_post_data_for_database();
        $this->model->create_new_record($data);
        // highlight-next-line
        set_flashdata('Record created successfully');
        redirect('tasks/manage');
    }
}
?>

Example #2: Using the Module Directly

You may also call the module method directly via Modules::run() if you prefer explicit routing.

<?php
Modules::run('flashdata/set_flashdata', 'Your changes have been saved');
?>

Notes

  • One at a time: Only a single flash message can be stored. A second call overwrites the first.
  • Redirect-friendly: Flashdata survives redirects because it is stored in the session, not in request data.
  • No formatting: The message is stored as plain text. Use the flashdata() method or helper to apply HTML wrapping when displaying.
  • Race condition awareness: If set_flashdata() is called but flashdata() is never called on the next page load, the message will remain in the session until the next call to flashdata() retrieves it.
