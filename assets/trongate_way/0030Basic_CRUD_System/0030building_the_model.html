Building the Model

The model handles all database interactions for the countries module. In Trongate v2, models extend the Model class and are stored in the module directory as Countries_model.php.

The Complete Model

<?php
class Countries_model extends Model {

    private string $table_name = 'countries';

    public function get_data_from_post(): array {
        return [
            'country_title' => post('country_title', true),
            'country_code' => post('country_code', true)
        ];
    }

    public function get_data_from_db(int $update_id): array {
        $record_obj = $this->db->get_where($update_id, $this->table_name);

        if ($record_obj === false) {
            http_response_code(404);
            echo 'Country record not found';
            die();
        }

        return (array) $record_obj;
    }

    public function fetch_records(int $limit, int $offset): array {
        $rows_to_return = (int) $limit;
        $rows_to_skip = (int) $offset;

        $sql = 'SELECT * FROM ' . $this->table_name
             . ' ORDER BY country_title ASC'
             . ' LIMIT ' . $rows_to_return
             . ' OFFSET ' . $rows_to_skip;

        return $this->db->query($sql, 'object');
    }

    public function prepare_records_for_display(array $rows): array {
        $prepared = [];
        foreach ($rows as $row) {
            $row_array = (array) $row;
            $prepared[] = (object) $this->prepare_for_display($row_array);
        }
        return $prepared;
    }

    public function prepare_for_display(array $data): array {
        return $data;
    }

    public function count_all(): int {
        return $this->db->count($this->table_name);
    }

}

Method by Method

get_data_from_post()

Retrieves submitted form data and sanitises it. The true parameter passed to post() tells Trongate to strip HTML tags and encode special characters. This method is called in two places: when displaying an empty create form (no previous data), and in the submit() method to build the array for database insertion or update.

get_data_from_db(int $update_id)

Fetches a single record by its primary key using $this->db->get_where(). If no record is found it sends a 404 response and terminates - this ensures that your views never receive an empty data set.

fetch_records(int $limit, int $offset)

Returns a paginated set of records sorted alphabetically by country name. The LIMIT and OFFSET values come from the controller's get_limit() and get_offset() helpers, which in turn read the current page number from the URL segment and the user's per-page preference from the session.

prepare_records_for_display() and prepare_for_display()

These methods let you transform raw database data before sending it to the view. In this simple example they pass the data through unchanged, but in a real-world module you might use them to format dates, join related data, or compute derived values. The two-method pattern - one for single records, one for collections - keeps your code clean and reusable.

count_all()

Returns the total number of records. The controller's pagination helper calls this to calculate how many pages are needed.

Naming Convention

Remember that the model file must be named after the module: Countries_model.php (note the singular "Country" becomes "Countries" for the module name, and the model appends _model). The class inside the file must match the filename: class Countries_model extends Model.
