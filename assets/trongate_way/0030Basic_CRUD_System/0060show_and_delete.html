Viewing and Deleting Records

Every CRUD module needs a way to view individual records in detail and a safe deletion flow that asks for confirmation before removing data. This page covers both.

The Show Method

The show() method displays a single country record. It retrieves the record ID from URL segment 3 and passes the data through prepare_for_display() before rendering. 

public function show(): void {
    $this->gatekeeper();

    $update_id = segment(3, 'int');

    if ($update_id === 0) {
        $this->not_found();
        return;
    }

    $data = $this->model->get_data_from_db($update_id);
    $data = $this->model->prepare_for_display($data);

    $data['update_id'] = $update_id;
    $data['headline'] = 'Country Details';
    $data['back_url'] = $this->get_back_url();
    $data['view_module'] = 'countries';
    $data['view_file'] = 'show';

    $this->templates->admin($data);
}

If the ID in the URL is 0 (meaning no valid ID was provided), the not_found() helper renders a polite "Country Not Found" page instead.

The Delete Confirmation

Deleting records requires two deliberate steps. First, the user visits confirm_delete() which asks "Are you sure?". Only after confirming does the actual deletion happen. 

public function confirm_delete(): void {
    $this->gatekeeper();

    $update_id = segment(3, 'int');
    $this->model->get_data_from_db($update_id);

    $data = [
        'headline' => 'Delete Country Record',
        'cancel_url' => BASE_URL . 'countries/show/' . $update_id,
        'form_location' => str_replace(
            '/confirm_delete',
            '/submit_confirm_delete',
            current_url()
        ),
        'update_id' => $update_id,
        'view_module' => 'countries',
        'view_file' => 'confirm_delete'
    ];

    $this->templates->admin($data);
}

public function submit_confirm_delete(): void {
    $this->gatekeeper();

    $update_id = (int) post('update_id', true);
    $this->db->delete($update_id, 'countries');
    set_flashdata(
        'The country record was successfully deleted.'
    );
    redirect('countries/manage');
}

Notice that confirm_delete() calls get_data_from_db() before rendering. This serves as a safety check - if the record does not exist (someone bookmarked a stale URL, for example), the model will return a 404 response and terminate before any confirmation form is displayed.

The Delete Confirmation View

<h1><?= $headline ?></h1>
<div class="card">
    <div class="card-heading">Confirmation Required</div>
    <div class="card-body text-center">
        <h4>Are you sure?</h4>
        <p>You are about to delete a country record.
           This cannot be undone. Do you really want to do this?</p>

        <?php
        echo form_open($form_location);
        echo '<div class="text-center">';
        echo anchor($cancel_url, 'Cancel',
            array('class' => 'button alt'));
        echo form_submit('submit', 'Yes - Delete Now',
            array('class' => 'danger'));
        echo form_hidden('update_id', $update_id);
        echo form_close();
        echo '</div>';
        ?>
    </div>
</div>

The form has two buttons: Cancel (an anchor link back to the show page), and Yes - Delete Now (the red submit button). Only the submit button triggers the actual deletion. The update_id is passed as a hidden field so submit_confirm_delete() knows which record to remove.

The Not Found Page

For completeness, here is the not-found view that appears when an invalid or missing record ID is provided: 

<h1><?= $headline ?></h1>
<div class="card">
    <div class="card-body text-center">
        <h4><?= $message ?? 'Record not found.' ?></h4>
        <div class="mt-3">
            <?= anchor($back_url, 'Go Back',
                array('class' => 'button alt')) ?>
        </div>
    </div>
</div>

This pattern - validate existence early, provide clear navigation, use flashdata for feedback - ensures a robust and user-friendly experience throughout the CRUD lifecycle.
