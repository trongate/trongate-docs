Creating and Editing Records

Creating and editing records follows the same form-and-submit pattern used throughout Trongate. A single create() method renders the form (empty for new records, pre-populated for edits), and a submit() method handles validation and persistence.

The Create Method

public function create(): void {
    $this->gatekeeper();

    $update_id = segment(3, 'int');
    $submit = post('submit');

    if (($update_id === 0) || ($submit === 'Submit')) {
        $data = $this->model->get_data_from_post();
    } else {
        $data = $this->model->get_data_from_db($update_id);
    }

    $data['headline'] = ($update_id === 0)
        ? 'Create Country Record'
        : 'Update Country Record';
    $data['update_id'] = $update_id;
    $data['form_location'] = str_replace('/create', '/submit', current_url());
    $data['view_module'] = 'countries';
    $data['view_file'] = 'create';

    $this->templates->admin($data);
}

This method handles three distinct situations:

  • Creating a new record: Visiting /countries/create gives update_id of 0, so it calls get_data_from_post() which returns empty strings. The form renders blank.
  • Editing an existing record: Visiting /countries/create/5 gives update_id of 5. It calls get_data_from_db(5) to pre-populate the form with the current values.
  • Re-displaying after validation failure: When submit() detects invalid data, it calls $this->create() again. This time $submit equals 'Submit', so it uses get_data_from_post() to preserve the user's submitted values rather than the database values.

The Create View

<h1><?= $headline ?></h1>
<div class="card">
    <div class="card-heading">Country Details</div>
    <div class="card-body">
        <?php
        echo form_open($form_location);
        echo validation_errors();

        echo form_label('Country Name');
        echo form_input('country_title', $country_title,
            ["placeholder" => "Enter Country Name",
             "autocomplete" => "off"]);

        echo form_label('Country Code');
        echo form_input('country_code', $country_code,
            ["placeholder" => "e.g. GB, FR, DE",
             "maxlength" => "2",
             "autocomplete" => "off"]);

        echo '<div class="text-center">';
        echo anchor('countries/manage', 'Cancel',
            array('class' => 'button alt'));

        if ($update_id > 0) {
            echo anchor(
                'countries/confirm_delete/' . $update_id,
                'Delete Country',
                array('class' => 'button danger')
            );
        }

        echo form_submit('submit', 'Submit');
        echo '</div>';

        echo form_close();
        ?>
    </div>
</div>

Notice the Delete Country link only appears when editing an existing record - it would make no sense on the create form. The form_open() helper uses the $form_location URL, which points to /countries/submit (or /countries/submit/5 for updates).

The Submit Method

public function submit(): void {
    $this->gatekeeper();

    $this->validation->set_rules(
        'country_title', 'Country Name',
        'required|min_length[2]|max_length[100]'
    );
    $this->validation->set_rules(
        'country_code', 'Country Code',
        'required|min_length[2]|max_length[2]'
    );

    $result = $this->validation->run();

    if ($result === true) {
        $data = $this->model->get_data_from_post();
        $update_id = segment(3, 'int');

        if ($update_id === 0) {
            $this->db->insert($data, 'countries');
            set_flashdata(
                'The new country record was successfully created.'
            );
        } else {
            $this->db->update($update_id, $data, 'countries');
            set_flashdata(
                'The country record was successfully updated.'
            );
        }

        redirect('countries/manage');
    } else {
        $this->create();
    }
}

Key points in this method:

  • Validation rules are set using set_rules() with the familiar pipe-delimited format. The country name must be 2–100 characters, and the country code must be exactly 2 characters.
  • If validation passes: the data is inserted or updated, a flashdata message is set, and the user is redirected back to the manage page.
  • If validation fails: the method calls $this->create() instead of rendering a view directly. This re-displays the form with validation errors shown by validation_errors() in the view, and preserves the user's submitted input.

Flashdata

The set_flashdata() function stores a one-time message in the session that is displayed on the next page load. In the manage view, flashdata() renders this message. After the page is shown, the message is automatically cleared from the session.
